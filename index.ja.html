全体のたとえ

TPは森のようなものです。XMLは木を植え、XSDは形を確認し、XSLTは読める地図に変換し、RDFは道をつなげ、RDFaはエージェント用の見えないラベルを付けます。

TP1 : XML、DTD、XPath、CSS、SVG

TP1では基礎を学びました。XML文書は木構造であり、DTDで検証でき、XPathで検索でき、SVGで図形も表現できます。

学んだこと

  • 整形式XMLには1つのルートと正しい入れ子構造が必要です。
  • 妥当なXMLはDTDなどの文法にも従います。
  • XPathはXMLの木に質問する方法です。
  • SVGはXMLで画像を表現できることを示します。

たとえ : XMLは木のようなものです。根が全体を支え、枝が情報を整理し、葉がデータを持ちます。

森のDTD

<!ELEMENT foret (#PCDATA|arbre)*>
<!ELEMENT arbre (tronc)>
<!ELEMENT tronc (branche*)>
<!ELEMENT branche (branche|feuille)*>
<!ELEMENT feuille EMPTY>

TP2 : XML Schema と XSLT

このTPではXMLをさらに深く学びました。DTDとXSDでデータを検証し、XSLTでHTMLに変換します。

重要点

  • XSDはDTDより細かい型を定義できます。
  • XSLTはテンプレートでXML要素をHTMLに変換します。
  • CSSはXSLTが生成するHTML内で使います。

たとえ : XSDは品質チェック担当、XSLTは内部データを公開ページに変換する翻訳者です。

変換例

<xsl:template match="/">
  <html><body><xsl:apply-templates/></body></html>
</xsl:template>

TP4 : XML Schema、XSLT、Unicode

このTPでは欧州議会の討論XMLとUnicodeエンコーディングを扱いました。

XML Schema

国、政治グループ、議員、委員、会議、発言を表現し、キー制約で参照の正しさを確認します。

<xs:key name="cleDeputes">
  <xs:selector xpath=".//depute"/>
  <xs:field xpath="@id"/>
</xs:key>

たとえ : XML Schemaは入場ルールのようなものです。

Unicode

UCS-4、UTF-8、UTF-16 BEで文字がどう表現されるかを確認しました。

たとえ : Unicodeは文字の世界共通パスポートです。

TP5 : RDFa と セマンティックWeb

TP5ではHTMLページにRDFa注釈を追加し、人間にもロボットにも読めるページを作りました。

HTML内のRDFa

about、typeof、property、rel、resource でHTMLからRDFトリプルを作れます。

<section about="#hasina" typeof="schema:Person foaf:Person">
  <h1 property="schema:name foaf:name">Hasina RAKOTOARISON</h1>
</section>

たとえ : RDFaはページに貼る見えない付箋です。

LODと抽出

FOAF、Dublin Core、Schema.org、DBpediaを使い、owl:sameAs でLinked Open Dataと接続します。
