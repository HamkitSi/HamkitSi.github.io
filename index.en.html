Lab Work

This section gathers the Semantic Web labs. I do not only link the PDF reports : I also summarize the important concepts, the produced files and the knowledge gained in each lab.

Learning path

The progression is : structure data with XML, validate it with DTD/XSD, transform it with XSLT, then connect it with RDF, SPARQL and RDFa.

General analogy

The labs are like a forest : XML plants the trees, XSD checks their shape, XSLT turns them into a readable map, RDF links paths together, and RDFa adds invisible labels for agents.

TP1 : XML, DTD, XPath, CSS and SVG

TP1 introduces the basics : an XML document is a tree, malformed files can be corrected, a DTD can validate the structure, XPath can query XML, and SVG can create visuals.

What I learned

  • A well formed XML document has one root and correctly nested tags.
  • A valid XML document follows a grammar such as a DTD.
  • XPath asks questions to the XML tree.
  • SVG shows that XML can also describe images.

Analogy : XML is like a tree : the root holds everything, branches organize information, and leaves contain data.

One DTD for the forest

<!ELEMENT foret (#PCDATA|arbre)*>
<!ELEMENT arbre (tronc)>
<!ELEMENT tronc (branche*)>
<!ELEMENT branche (branche|feuille)*>
<!ELEMENT feuille EMPTY>

TP2 : XML Schema and XSLT

This lab goes deeper into XML : DTD and XSD control the data, then XSLT turns the document into readable HTML.

Key points

  • XSD defines richer types than a DTD : strings, identifiers, references, dates and complex structures.
  • XSLT uses templates : when an XML element is matched, it is transformed into HTML.
  • CSS is linked in the HTML generated by XSLT.

Analogy : XSD is a quality controller, XSLT is a translator that turns internal records into public pages.

Transformation example

<xsl:template match="/">
  <html><body><xsl:apply-templates/></body></html>
</xsl:template>

TP3 : RDF, SPARQL and Corese

The Ice Company universe is turned into an RDF graph. Data is no longer only stored as a tree : characters, cities, trains and companies are connected.

RDF modeling

The model uses four classes : Personnage, Ville, Train and Compagnie. Properties describe residence, employment, train routes and ownership.

:ExpressPolaire :relie :PortNord .

Analogy : RDF is like a metro map : stations are resources, lines are properties, and SPARQL asks for routes.

TP4 : XML Schema, XSLT and Unicode

This lab combines two topics : reliable XML for European Parliament debates and Unicode character encoding.

XML Schema part

The schema describes parliamentary debates with countries, political groups, members, commissioners, sessions and interventions. Keys ensure references point to existing elements.

<xs:key name="cleDeputes">
  <xs:selector xpath=".//depute"/>
  <xs:field xpath="@id"/>
</xs:key>

Analogy : XML Schema acts like entry rules : every speaker must have a valid identity before speaking in the document.

Unicode part

The application explains UCS-4, UTF-8 and UTF-16 BE. It highlights structure bits and data bits.

Analogy : Unicode is a universal passport for characters.

TP5 : RDFa and Semantic Web

TP5 shows how HTML can be enriched with RDFa annotations so that a page is readable by humans and exploitable by robots.

RDFa in HTML

Attributes such as about, typeof, property, rel and resource generate RDF triples from HTML.

<section about="#hasina" typeof="schema:Person foaf:Person">
  <h1 property="schema:name foaf:name">Hasina RAKOTOARISON</h1>
</section>

Analogy : RDFa is like invisible sticky notes on the page.

LOD and extraction

The lab uses FOAF, Dublin Core, Schema.org and DBpedia. owl:sameAs links local resources to Linked Open Data.
